# gitignore-discipline

The rules for what never leaves the local machine when a project is pushed to GitHub. The first entry is always `.env`: API keys and secrets that become visible to the internet if pushed to a public repo. Van Clief treats `.env` leakage as the most common beginner security mistake. Other standard exclusions: `node_modules` (rebuildable dependencies), build caches and `dist/` output, and private markdown like internal `CLAUDE.md` or brand-voice documents. The deciding question: "would I be comfortable if a stranger read this file?" If no, add it to `.gitignore`.

## Claims

- Must-exclude: `.env` files. Pushing them to a public repo exposes API keys and passwords [sources/playbooks-3-2-github-and-folder-structure]
- Must-exclude: `node_modules` (rebuildable dependency files, large, not needed in the repo) [sources/playbooks-3-2-github-and-folder-structure]
- Must-exclude: build cache and `dist/` folders (regenerated by GitHub Actions) [sources/playbooks-3-2-github-and-folder-structure]
- Optional-exclude: private markdown (internal `CLAUDE.md`, brand voice, notes). Use `*.md` or list specific filenames [sources/playbooks-3-2-github-and-folder-structure]
- Decision rule: "would I be comfortable if a stranger read this file?" If no, add it [sources/playbooks-3-2-github-and-folder-structure]
- `.env` leakage is named the most common beginner security mistake [sources/playbooks-3-2-github-and-folder-structure]
