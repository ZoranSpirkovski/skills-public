# folder-as-ui

A subclaim of [[folder-as-workspace]]: the folder structure itself functions as the application's user interface. Navigating folders, renaming files, moving artifacts between directories - these are the user actions of the system. No separate admin panel, configuration screen, or app UI is needed. The file explorer / IDE / Finder is the UI, and the user can drive it by keyboard, by click, or (Van Clief predicts) by voice.

## Claims

- "The folder becomes your app. This is your UI. What's simpler UI than a folder?" [sources/folder-system-transcript]
- The next stage of this idea: voice-driving the folder - talking to the folder structure as the primary input method [sources/folder-system-transcript]
- All UI actions (creating, renaming, moving files) are already fluent operations users know - no new interface to learn [sources/folder-system-transcript]
- VS Code is an optional convenience for navigating the folder UI; a plain file explorer with Notepad works just as well [sources/foundation-4-1-install-and-first-use]
