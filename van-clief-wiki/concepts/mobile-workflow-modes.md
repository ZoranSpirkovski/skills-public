# mobile-workflow-modes

Mobile work with Claude is a different mode, not a smaller version of desk work. Three modes. Mode 1 (check and acknowledge): 10-second status glance. Mode 2 (quick intervention): one or two short messages at a decision point. Mode 3 (voice-driven work): longer instructions spoken instead of typed. Mobile works for monitoring, binary decisions, short responses, voice input, approvals, reading outputs. Mobile does not work for deep editing, multi-file context, long-form writing, debugging, or nuanced creative direction. Mobile access is a capability, not an obligation; often the best mobile workflow is not checking at all.

## Claims

- Three modes. Mode 1: check and acknowledge (10 seconds). Mode 2: quick intervention (1-2 short messages). Mode 3: voice-driven work (longer spoken instructions) [sources/stack-2-3-mobile-workflow-patterns]
- Mode 2 examples: "proceed with that approach", "use blue not green", "skip that section for now", "stop, I'll fix when I'm back" [sources/stack-2-3-mobile-workflow-patterns]
- Voice input beats mobile typing. Both the Claude app and the phone keyboard accept voice [sources/stack-2-3-mobile-workflow-patterns]
- Works on mobile: monitoring, binary decisions, short responses, voice instructions, approving actions, reading outputs [sources/stack-2-3-mobile-workflow-patterns]
- Does not work on mobile: deep editing, multi-file context, long-form writing, debugging, nuanced creative direction [sources/stack-2-3-mobile-workflow-patterns]
- Voice input tips: speak in complete thoughts, use natural language, pause for punctuation, review before sending, keep it short [sources/stack-2-3-mobile-workflow-patterns]
- Mobile access is a capability, not an obligation. Often the best mobile workflow is not checking at all [sources/stack-2-3-mobile-workflow-patterns]
- Check when task has decision points Claude cannot make alone, or when ambiguous input needs course correction. Wait otherwise [sources/stack-2-3-mobile-workflow-patterns]
