# abstraction-ladder

Van Clief's three-level framework for figuring out where any AI investment sits on the commoditization curve. A story is a book for $15, identical for every reader. The same story as a movie costs hundreds of millions and the experience is bigger. As a video game it costs billions and each player's playthrough is unique. Same story, three layers, exponentially more value captured per layer because the experience got harder to replicate. Mapped onto AI work: Layer 1 is typing prompts and getting outputs (the book, commoditized), Layer 2 is building flows that connect prompts to specific business processes (the movie), Layer 3 is systems that generate their own data and use it to learn (the video game). The practical rule: give away whatever AI made cheap, charge for the bespoke layer above.

## Claims

- Three levels of AI use: typing prompts (book), building flows (movie), systems that learn from their own data (video game) [sources/foundation-2-4-the-ladder]
- Value always migrates above whatever just got commoditized [sources/foundation-2-4-the-ladder]
- "Don't automate what the next model will give you for free" - invest in what is specific to your context [sources/foundation-2-4-the-ladder]
- Case: a web agency gave away basic HTML generation (now near-free via tools like Lovable) and kept charging $20K/month for research, design strategy, and judgment [sources/foundation-2-4-the-ladder]
- Same pattern played out in programming languages: machine code → assembly → C → Python → frameworks → AI tools; each previous layer became infrastructure [sources/foundation-2-4-the-ladder]
- Automation kills specific middleman positions; underlying markets persist because human desire does [sources/foundation-2-4-the-ladder]
- The technology-stacking pattern in education (calculator, internet, AI) is the same structural claim as the ladder: each new layer moves the old expert skill to the baseline and pushes senior work higher [sources/archive-2-2-raise-the-bar]
- The coding pyramid (machine code, Python, LLMs) is the technical form of the ladder: each abstraction layer lets a novice do what used to require an expert; value accrues above the newest layer [sources/archive-2-2-raise-the-bar] [sources/archive-3-4-computational-orchestration]
