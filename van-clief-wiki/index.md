## Concepts

### Core folder methodology (the spine)

- [[three-layer-routing]] - map (CLAUDE.md) → rooms (CONTEXT.md) → tools (skills/MCP)
- [[routing-table]] - the task → go-to → read → skills table; the system's most important pattern
- [[context-md-workspace-files]] - Layer 2 per-workspace context files, loaded on entry
- [[claude-md-three-file-pattern]] - the minimum-viable entry folder
- [[naming-conventions-as-db]] - predictable filenames replace databases and query layers
- [[skills-wired-per-task]] - skills referenced from routing table rows, not loaded globally
- [[folder-as-workspace]] - the folder IS the app; no framework required
- [[folder-as-ui]] - the folder structure functions as the user interface
- [[workspace-isolation]] - sibling workspaces don't bleed context into each other
- [[folder-level-readme]] - per-folder context files that scale Layer 2 beyond shallow trees

### Foundational mechanics

- [[context-window]] - the finite token budget that drives routing discipline
- [[tokens]] - the unit of measurement for context window budget
- [[markdown]] - the plain-text format of every file in the system
- [[three-interfaces]] - Desktop (planning), VS Code (editing), terminal (execution) over one model
- [[prompt-framework-5-part]] - Identity, Task, Context, Constraints, Output Format
