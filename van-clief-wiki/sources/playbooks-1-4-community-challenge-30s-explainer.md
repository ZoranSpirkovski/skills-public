# playbooks-1-4-community-challenge-30s-explainer

> Source: Implementation Playbooks course, Module 1 Lesson 1.4. Written community-challenge page (no video). File: `.wiki/raw/playbooks/lesson-1-4-community-challenge-30s-explainer.md`.

## Summary

The community exercise that closes the animation module. Build a 30-second explainer on any topic using the script-spec-build-render pipeline taught in 1.1 through 1.3. The only rule is to use the workflow. The lesson supplies an "Explainer Brief" template with five fields (concept, audience, hook, explanation, payoff, visual style, reference) that the learner fills in and hands to Claude as input for the spec.

## Key claims

1. Thirty seconds is chosen deliberately: long enough to explain a real concept, short enough to finish in one session.
2. If you can explain a concept in 30 seconds with supporting visuals, you understand both the concept and the workflow. The challenge doubles as a comprehension test.
3. The starter brief has five fields: concept (one sentence), audience, hook (0:00-0:05), explanation (0:05-0:25 in 2-3 beats), payoff (0:25-0:30), and optional visual style / reference.
4. The brief is handed directly to Claude to generate the spec ("Here is my brief: [paste]. Write a spec for this animation.").
5. Four standout traits for featured builds: clarity (concept lands in 30s), craft (animations support the explanation), creativity (unexpected use of the workflow), honesty (sharing what went wrong).
6. "Share what you learned for next time" is called out as the single most valuable post-challenge contribution.
