# foundation-3-3-common-mistakes

> Packaged inside `.wiki/raw/the-foundation.md`, lines ~1689-1776. Foundation Module 3, Lesson 3.

## Summary

Seven recurring mistakes Van Clief has watched community members make when setting up the folder architecture, each with a fix. The pattern across all seven: keep the system small, focused on the work (not on Claude's personality), updated as projects evolve, and grown incrementally from real use rather than planned to completion in a weekend.

## Key claims

1. Mistake 1 - CLAUDE.md too long. Fix: one screen, under 40-50 lines; pull everything else into workspace context files.
2. Mistake 2 - skipping the routing table. Fix: three-column minimum (task, go-to, read); four if skills are wired.
3. Mistake 3 - too many workspaces. Fix: start with 2-3; split by mental mode, not by stage.
4. Mistake 4 - context files that describe Claude's personality instead of the work. Fix: flip the ratio to 80% about the work, 20% about behavior.
5. Mistake 5 - never updating context files. Fix: treat them like working notes; "Last updated" line is a simple nudge.
6. Mistake 6 - one flat folder with no subfolders. Fix: more than 8-10 files at one level requires subfolders.
7. Mistake 7 - building the whole system before using it. Fix: first version should take 15 minutes; add structure in response to real work, not anticipation.
8. Closing rule: if the folder system starts feeling heavy or complicated, something went wrong. Go back to the three layers - map, rooms, tools - and check each is doing only its job.
