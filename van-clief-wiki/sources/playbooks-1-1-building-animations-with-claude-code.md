# playbooks-1-1-building-animations-with-claude-code

> Source: Implementation Playbooks course, Module 1 Lesson 1.1. Video transcript plus written lesson. File: `.wiki/raw/playbooks/lesson-1-1-building-animations-with-claude-code.md`.

## Summary

The full animation workflow from script to rendered video. Van Clief introduces the two-workspace setup (Script Lab + Animation Studio) and the four-stage pipeline (script, spec, build, render). The spec is the load-bearing artifact: a plain-English markdown contract between voiceover and animation, covering beat map, visual philosophy, key moments, and audio sync points. The build uses [[remotion]] plus React to generate frames. Rendering produces an MP4 that goes into CapCut for final editing. The lesson positions this as the concrete application of `video-as-code` theory from Foundation 2.6.

## Key claims

1. The four stages are script, spec, build, render; AI can automate every stage or the user can intervene at any one.
2. The spec is a contract between the voiceover and the animation, making timing carry over from script to scenes.
3. A spec contains: beat map (timestamped), visual philosophy, key moments, audio sync points.
4. A spec does NOT contain: frame numbers, pixel positions, component props, code. Including those constrains Claude and produces worse animations.
5. The two-workspace setup uses a parent folder with Script Lab (long-form + short-form) and Animation Studio (specs, builds, output), linked by a root `CLAUDE.md` that routes between them.
6. Remotion renders React components to video frame-by-frame; Claude can install Remotion, Node.js, and the Remotion agent skills directly from a single prompt.
7. Remotion agent skills are markdown files plus code samples that Claude reads only when relevant (3D content, DOM elements, Tailwind, transitions). Skills are not loaded all at once.
8. The folder structure is the software; Claude Code is the runtime. Same result whether accessed via the desktop app, a VS Code integration, or terminal.
9. Giving Claude creative room within clear constraints produces better animation than tightly-specified pixel control. Connects to the inverted-U from Foundation 2.6.
10. Each scene is a separate React component file. Claude edits individual components on natural-language requests ("make that gauge bigger") without rewriting the whole pipeline.
11. Component libraries accumulate over time: text animations, backgrounds, transitions, data visualizations. Claude assembles reusable pieces instead of inventing each time (separation of concerns).
12. The rendered animation is a starting point, not a finished video. Voiceover, music, captions, and pacing get finished in CapCut or a similar editor.
13. A 30-second animation is a better first project than a 10-minute one. Get the workflow operational first, then scale.
14. The whole PowerPoint that accompanies this course was generated by Claude Code in four prompts using the same folder method.
