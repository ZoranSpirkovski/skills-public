# foundation-4-5-where-this-goes

> Packaged inside `.wiki/raw/the-foundation.md`, lines ~2207-2312. Foundation Module 4, Lesson 5 (Session 5 of 5).

## Summary

The scale-up session. One `CLAUDE.md` per project stops working once the user runs ten projects or one complex multi-modal workflow. Van Clief introduces task routing as the next level: different tasks automatically load different context. The three layers from Module 3 map onto the operational reality: Layer 1 `CLAUDE.md` as map, Layer 2 per-workspace `CONTEXT.md` as rooms, Layer 3 skills and tools plugged in per-workspace. Three scaling principles follow: one fact one location, new sessions start clean, and the system is hand-offable to a team. The lightest test: two separate folders with two distinct `CLAUDE.md` files; watch Claude behave differently in each.

## Key claims

1. One `CLAUDE.md` per project is a working system; the scale-up is task routing that loads different context per task automatically.
2. Van Clief's personal workspace: "write a script" loads voice docs, "build an animation" loads the design system and component library, "build a course module" loads curriculum and guidelines.
3. The three layers in operation: Layer 1 `CLAUDE.md` routes; Layer 2 workspace `CONTEXT.md` describes the room; Layer 3 skills and tools plug-and-play per workspace.
4. Three scaling principles: one fact lives in one location (no duplication across context files); new sessions start clean (each conversation re-reads the `CLAUDE.md` fresh); the system is hand-offable because context lives in files, not people's heads.
5. Context-window scarcity is the scaling motivation: 200K sounds huge until filled with irrelevant files. Task routing is the mechanism that keeps input clean.
6. Lightest possible test: two separate folders, each with its own `CLAUDE.md`; run a task in each and feel the difference.
7. The five-session arc closes by listing what the user has built: three interfaces, real tasks on their own files, Desktop/Code pairing, a project-level CLAUDE.md, and basic two-folder routing.
8. Module 3 (specifically [[foundation-3-1-full-walkthrough]]) is named as the next step for anyone who wants the full system beyond the five-session minimum.
