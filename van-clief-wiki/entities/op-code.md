# op-code

An open-source [[claude-code]] wrapper focused on project management: usage tracking, multi-project switching, cleaner interface than raw terminal. Studied as reference during Van Clief's Building Your Stack front-end build. Van Clief judged it too feature-heavy for his needs but kept the underlying patterns.

## Claims

- Another Claude Code wrapper, oriented around project management rather than raw chat [sources/stack-1-4-repo-tour-open-source-references]
- Worth studying for usage tracking implementation, multi-project switching, and config system structure [sources/stack-1-4-repo-tour-open-source-references]
- Van Clief did not use most of its UI complexity: too many features for his needs. But the underlying patterns are solid [sources/stack-1-4-repo-tour-open-source-references]
- Surveyed alongside [[claude-code-web]] during the initial research phase [sources/stack-1-2-how-i-use-claude-code-in-build]
- Van Clief's build fetched specific files from it rather than cloning it whole [sources/stack-1-3-designing-for-your-use-case]
