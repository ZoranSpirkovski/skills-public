# grace-hopper

American computer scientist and US Navy Rear Admiral. In 1952 she built a working compiler that could translate human-readable instructions into machine code - and nobody would touch it. "They told me computers could only do arithmetic." Van Clief cites Hopper repeatedly as the template for every new abstraction layer: each one arrives untrusted, faces skepticism from people pointing at the layer below, and eventually becomes the infrastructure that enables the next layer.

## Claims

- Built a working compiler in 1952 that translated closer-to-English instructions into machine code [sources/foundation-2-3-1953-word-game]
- "I had a running compiler and nobody would touch it. They told me computers could only do arithmetic." [sources/foundation-2-3-1953-word-game]
- Said "the most dangerous phrase in the language is 'we've always done it this way'" [sources/foundation-2-3-1953-word-game]
- Cited as the precedent for the skepticism-then-infrastructure pattern that recurs at every new abstraction layer including AI [sources/foundation-2-3-1953-word-game] [sources/foundation-2-7-nazi-psychology-ai-auditing]
- Coined the term "bug" (or at least popularized it after the 1947 Mark II moth incident) [sources/foundation-2-2-one-line-of-python]
